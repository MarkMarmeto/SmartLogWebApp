Features

SmartLog Web App

School Information Management System for Philippine K-12 schools

User & Access Students Enrollment Attendance SMS Calendar Devices Reports Settings System
SmartLog runs on the school's local network (LAN-only, offline-capable) and serves as the central hub for attendance, notifications, and records. It is designed for Philippine K-12 schools with support for bilingual SMS notifications and GSM modem-based messaging.
🔐
User & Access Management
  • Role-based access control: SuperAdmin, Admin, Teacher, Security, Staff
  • Secure login with account lockout (5 attempts → 15-min lockout)
  • 10-hour sliding sessions (HttpOnly, Secure, SameSite cookies)
  • Per-user profile with profile picture upload
  • Password change with forced change on first login
  • User activation/deactivation by admins
  • Full audit log of user actions (who, what, from which IP)
  • CSRF protection via anti-forgery tokens
👤
Student Management
  • Add, edit, deactivate, and search students
  • Auto-generated Student ID: YYYY-GG-NNNN
  • LRN (Learner Reference Number) tracking
  • Assign to grade level and section
  • Parent/guardian mobile number for SMS
  • Per-student SMS language preference (EN / Filipino)
  • Bulk import students via CSV upload
  • Profile picture upload per student
  • QR code generation (HMAC-SHA256 signed, printable)
  • QR code regeneration/invalidation (old code immediately revoked)
📚
Academic Year & Enrollment
  • Define and manage academic years (e.g., 2025-2026)
  • Mark one year as current; historical years remain accessible
  • Manage grade levels (K-12) and sections (name, capacity, adviser)
  • Annual batch re-enrollment: promote, graduate, or skip students
  • Per-student active enrollment per academic year
📋
Attendance Tracking
  • QR code scanning via registered scanner devices
  • ENTRY and EXIT scan types
  • Duplicate prevention (5-minute window per student/type)
  • Calendar-aware: rejects scans on holidays and suspensions
  • Attendance summary dashboard (by grade, section, date)
  • Daily attendance list with search and filter
  • Attendance trend chart (past 30 days)
  • Attendance by grade chart
  • Recent activity feed
💬
SMS Notifications
  • Automated SMS to parents on student entry/exit
  • Bilingual templates: English and Filipino per student
  • Template codes: ENTRY, EXIT, HOLIDAY, SUSPENSION, EMERGENCY
  • Customizable templates via admin UI
  • SMS broadcast: send announcements to filtered groups
  • Schedule broadcasts for future delivery
  • Cancel pending scheduled broadcasts
  • SMS queue with priority ordering and ScheduledAt support
  • Retry with exponential backoff (2→4→8 min, max 3 retries)
  • GSM Modem gateway (USB serial, offline-capable, ~₱1/SMS)
  • Semaphore cloud gateway (internet required)
  • Automatic fallback from GSM Modem to Semaphore
  • SMS delivery reports via webhook (Semaphore)
  • Full SMS history log with delivery status
📅
Calendar Management
  • School calendar with events, holidays, and suspensions
  • 13 Philippine national holidays pre-loaded per academic year
  • Mark events as "Affects Attendance" to block scans
  • Grade-specific events (e.g., Grade 12 activity only)
  • Event types: Holiday, School Event, Class Suspension
🖥️
Scanner Device Management
  • Register scanner devices and generate API keys
  • API key shown once; stored as SHA-256 hash
  • Activate/deactivate devices; revoked devices immediately rejected
  • View last-seen timestamp per device
  • Multiple devices supported (one API key each)
📊
Reporting & Export
  • Daily Attendance Report — export CSV or HTML
  • Weekly Attendance Report — export CSV
  • Monthly Attendance Report — export CSV
  • Student Attendance Report (individual history) — export CSV
  • Audit Log Export (SuperAdmin only) — export CSV
⚙️
Settings & Configuration
  • Runtime configuration via Admin > Settings (no restart required)
  • SMS provider switch, API key updates, test SMS
  • GSM modem port and baud rate configuration
  • HMAC secret rotation (invalidates all existing QR codes)
  • Application timezone setting
  • QR code settings
🔧
System & Security
  • Automatic database migration on startup (EF Core)
  • Seed data on first run: grade levels, sections, admin accounts, holidays
  • CORS configuration for browser-based scanner clients
  • HTTPS-ready (configure SSL certificate in production)
  • Docker deployment (docker-compose with SQL Server container)
  • Windows Service deployment (auto-start on boot, auto-restart)
  • Structured logging via Serilog (file + console)
  • GitHub Actions CI (build + test on every push)
  • GitHub Actions CD (release on version tags)
