Part 1: GSM SIM Module vs. Cloud SMS

SmartLog currently supports a GSM modem gateway (serial port AT commands) for offline SMS delivery, with Semaphore as a cloud fallback. This section compares the two fundamental approaches.

On-Premise

GSM SIM Module

A hardware modem (SIM800L, SIM7600, Quectel EC25, or USB dongle) with a registered prepaid/postpaid SIM card, connected directly to the SmartLog server via serial port or USB.

How it works: The server sends AT commands over serial to instruct the modem to send SMS via the cellular network. No internet required.

Cloud API

Cloud SMS Provider

A third-party service (Semaphore, Twilio, etc.) with a REST API. The SmartLog server sends HTTP requests to the provider's API, which routes the SMS through telco-grade infrastructure.

How it works: The server calls the provider's API over the internet. The provider handles routing, delivery, and compliance.

Detailed Pros & Cons

GSM SIM Module

Advantages

  • Works without internet — ideal for remote schools with unreliable connectivity
  • Very low per-message cost when using prepaid promo packages
  • No recurring API subscription fees
  • Full control over the hardware and sending process
  • No third-party dependency for core SMS delivery
  • Data stays on-premise (no external API calls with student info)

Disadvantages

  • Throughput bottleneck: At ~7 SMS/min, a school with 500 students takes over an hour to send all notifications
  • 2G sunset: SIM800L/SIM900 modules will stop working as Globe/Smart decommission 2G. Must invest in 4G modules (SIM7600/Quectel EC25)
  • SIM blocking risk: Using consumer prepaid promos for bulk SMS violates telco terms of service; SIM may be blocked
  • Hardware failures: Modem overheating, serial port issues, power supply problems
  • SIM management: Must keep SIM loaded with credits/promos, ensure SIM registration remains valid
  • No professional sender ID: Parents receive SMS from an unknown phone number
  • Cross-network cost: Globe SIM sending to Smart costs more than same-network
  • No delivery confirmation: Cannot reliably know if SMS was received

Cloud SMS API

Advantages

  • High throughput: Send hundreds of SMS per minute — all parents notified within seconds
  • All networks covered: Single API call reaches Globe, Smart, TNT, TM, Sun, DITO
  • Professional sender ID: Parents see "SmartLog" instead of random number
  • Reliable delivery reports: Know exactly which SMS was delivered, failed, or pending
  • Zero hardware maintenance: No modules, no SIMs, no serial ports
  • NTC compliance handled: Provider manages regulatory requirements
  • Easy to scale: 100 students to 10,000 students with no infrastructure changes
  • 99.5–99.9% uptime SLAs: Enterprise-grade reliability

Disadvantages

  • Requires internet: If school internet goes down, SMS cannot be sent
  • Per-message cost: PHP 0.25–0.50 per SMS adds up at volume
  • Recurring expense: Must budget for monthly SMS costs
  • Third-party dependency: Provider outage = no SMS
  • Data leaves premises: Student names and parent phone numbers sent to provider's API
SmartLog's Hybrid Approach SmartLog currently supports both GSM modem (primary) and cloud API (fallback). This is a valid strategy: the GSM modem handles day-to-day offline sending, while the cloud API kicks in as a fallback when the modem fails or for high-volume bursts. However, as the school grows, transitioning fully to cloud SMS is recommended.

Part 3: Detailed Provider Profiles

Local Providers (Recommended for PH Schools)

PHP Billing All PH Networks REST API Direct Telco Routes

Semaphore

The most popular SMS API provider among Filipino developers. Simple, affordable, and purpose-built for the Philippine market.

  • Pricing: ~PHP 0.35–0.50/SMS, prepaid credits starting at ~PHP 500
  • API: Simple REST API (HTTP GET/POST). Easy integration with any language including .NET HttpClient
  • Sender ID: Custom alphanumeric sender ID (e.g., "SmartLog") with approval
  • Delivery Reports: Supported via API polling or webhook callbacks
  • Coverage: Globe, Smart, TNT, TM, Sun, DITO — all major networks
  • Support: Email support, Filipino-language documentation, active developer community
  • Best for: Philippine startups, schools, SMEs needing simple, affordable SMS
PHP Billing All PH Networks REST + SMPP Lowest Cost

M360

A Philippine SMS aggregator with direct telco connections. Potentially the lowest per-SMS cost among local providers.

  • Pricing: ~PHP 0.25–0.50/SMS depending on volume tier
  • API: REST API and SMPP (telco-grade protocol) support
  • Sender ID: Custom sender ID available
  • Coverage: All Philippine networks with direct Globe and Smart connections
  • Best for: Cost-sensitive deployments, high-volume senders
PHP Billing All PH Networks Voice + SMS Cebu-based

Engagespark

Cebu-based platform focused on engagement campaigns. Offers both SMS and voice (IVR), popular with NGOs and educational institutions.

  • Pricing: ~PHP 0.40–0.75/SMS, voice calls also available
  • API: REST API + no-code flow builder for engagement campaigns
  • Unique: Airtime rewards, two-way SMS, IVR voice calls
  • Best for: Schools wanting SMS + voice notification capabilities
PHP Billing Globe/TM Only Official Telco API

Globe Labs

Globe Telecom's official developer API. Direct network access but limited to Globe and TM subscribers only.

  • Pricing: ~PHP 0.35–1.00/SMS
  • Critical Limitation: Only reaches Globe and TM subscribers. Would need a second provider for Smart/TNT/Sun/DITO
  • Best for: Scenarios where you know all recipients are Globe subscribers

International Providers

USD Billing All Networks .NET SDK Manila Office

Infobip

Enterprise-grade international provider with a Manila office and direct Philippine telco connections. Offers omnichannel (SMS + Viber + WhatsApp).

  • Pricing: ~USD 0.02–0.05/SMS (negotiable at volume)
  • Unique: Viber Business Messages integration — Viber is extremely popular in the Philippines and messages are free to receive
  • Best for: Larger schools wanting enterprise-grade reliability and Viber as a cost-saving channel
USD Billing All Networks Official .NET SDK Best Documentation

Twilio

The global leader in cloud communications. Best-in-class API documentation and developer experience, but expensive for Philippine SMS.

  • Pricing: ~USD 0.04–0.06/SMS to PH (~PHP 2.20–3.40)
  • Strengths: Unmatched documentation, official .NET SDK, Studio visual builder, global reliability
  • Weakness: 5–7x more expensive than local providers for PH SMS
  • Best for: Teams already on Twilio, or multi-country deployments
USD Billing All Networks REST API

Vonage (Nexmo) / MessageBird (Bird) / ClickSend

Other international providers with similar pricing and capabilities to Twilio. All route through international aggregators to reach Philippine networks.

  • Pricing: ~USD 0.03–0.07/SMS to PH (~PHP 1.70–3.90)
  • Not recommended for PH school deployments due to cost and lack of local presence

Part 5: Regulatory Considerations

SIM Registration Act (RA 11934)

Impact on GSM Modem Deployments The SIM Registration Act (enacted 2022) requires all SIM cards to be registered with valid government IDs. For schools using GSM modems:
  • SIM cards must be registered under the school's authorized representative
  • Unregistered SIMs are deactivated — your modem stops working
  • Using personal SIMs for institutional purposes may create compliance issues
  • Cloud SMS providers handle this entirely on their end

NTC Sender ID Registration

The National Telecommunications Commission (NTC) requires bulk/commercial SMS senders to register their sender IDs. Key points:

  • Unregistered alphanumeric sender IDs may be blocked or filtered by telcos
  • Cloud SMS providers like Semaphore and M360 handle NTC registration for you
  • GSM modems send from a regular phone number (no alphanumeric sender ID possible)
  • Parents are more likely to read SMS from "SmartLog" than from an unknown 09XX number

Data Privacy Act (RA 10173)

When sending student and parent information to cloud SMS providers:

  • Ensure the provider has appropriate data processing agreements
  • Only send the minimum necessary data (phone number + message text)
  • SmartLog already sends only phone number and pre-rendered message text — no raw student data
  • Local providers (Semaphore, M360) store data within Philippine jurisdiction
  • International providers may process data outside PH — check their DPA compliance

2G Network Sunset

Critical Risk for GSM Modem Users Globe and Smart are progressively decommissioning 2G (GSM) networks. The popular SIM800L and SIM900 modules are 2G-only and will stop working as 2G towers are shut down. If you continue using GSM modems, invest in 4G LTE modules:
  • SIM7600: ~PHP 1,500–3,000 (4G LTE, AT command compatible)
  • Quectel EC25: ~PHP 1,500–2,500 (4G LTE, good Linux/USB support)
  • Avoid: SIM800L (~PHP 150–300) and SIM900 (~PHP 300–500) for new deployments

Final Recommendation

For SmartLog's school attendance notification system deployed in Philippine schools:

Primary: Semaphore

Best overall balance of cost, simplicity, reliability, and Philippine market fit.
PHP-peso billing, all networks, simple REST API, proven track record with Filipino developers.

Alternative: M360 (lower cost) • Infobip (enterprise + Viber)

Keep the GSM modem as an offline fallback — SmartLog's hybrid architecture is sound. Use cloud SMS as the primary delivery channel for reliability, throughput, and professional sender ID. The GSM modem remains valuable for schools with unreliable internet, sending critical notifications when connectivity is down.

SmartLog Already Supports This Architecture SmartLog's SMS subsystem is already designed with a primary provider (GSM_MODEM) and fallback (SEMAPHORE). Switching the default to a cloud provider and keeping GSM as fallback requires only a configuration change in the admin SMS settings — no code changes needed.
