SmartLog Web App — Windows Setup Guide

Complete step-by-step guide to deploy the SmartLog Web Application on a Windows-based school server.

Automated Setup Available! For a faster installation, use the PowerShell setup wizard: 
# Open PowerShell as Administrator, navigate to the project folder, then run:
.\deploy\Setup-SmartLog.ps1
The script automates database creation, credentials generation, firewall rules, and Windows Service registration. Read this guide for manual setup or if the script encounters issues.

1. Prerequisites & Downloads

Download and install these on the Windows machine that will act as the school server:

SoftwareVersionDownload
.NET 8.0 SDK 8.0 or later dotnet.microsoft.com
SQL Server Express 2019 or later microsoft.com/sql-server
SQL Server Management Studio (SSMS) Latest learn.microsoft.com
Git for Windows Latest git-scm.com
Hardware Recommendations: Windows 10/11 Pro or Windows Server 2019+, at least 4 GB RAM, 20 GB free disk space. The server should be connected to the school LAN via Ethernet (not Wi-Fi) for reliability.

2. Install Required Tools

1 Install .NET 8.0 SDK

  1. Download the .NET 8.0 SDK installer (not Runtime) for Windows x64
  2. Run the installer and follow the prompts
  3. Open Command Prompt or PowerShell and verify:
dotnet --version

You should see a version starting with 8.0 (e.g., 8.0.401).

2 Install SQL Server Express

  1. Download SQL Server 2022 Express (free edition)
  2. Choose "Basic" installation type
  3. Note the instance name — default is SQLEXPRESS
  4. Note the connection string shown at the end of installation
  5. Enable TCP/IP protocol:
    • Open SQL Server Configuration Manager
    • Navigate to SQL Server Network Configuration → Protocols for SQLEXPRESS
    • Right-click TCP/IP → Enable
    • Right-click TCP/IP → Properties → IP Addresses tab → scroll to IPAll → set TCP Port to 1433
    • Restart the SQL Server service

3 Install SSMS (SQL Server Management Studio)

  1. Download and install SSMS
  2. Open SSMS and connect to localhost\SQLEXPRESS using Windows Authentication
  3. Verify you can see the server in the Object Explorer

4 Install Git for Windows

  1. Download and run the Git installer
  2. Use the default options during installation
  3. Verify in Command Prompt:
git --version

3. Clone the Repository

Open Command Prompt or PowerShell and clone the project:

cd C:\
git clone https://github.com/MarkMarmeto/SmartLogWebApp.git
cd SmartLogWebApp
You can clone it to any folder. We use C:\SmartLogWebApp in this guide for simplicity.

4. Set Up SQL Server Database

1 Create the Database

Open SSMS, connect to localhost\SQLEXPRESS, and run:

CREATE DATABASE SmartLogDb;
GO

2 Create a SQL Login (recommended for production)

Instead of using Windows Authentication, create a dedicated SQL login:

USE [master];
GO

CREATE LOGIN SmartLogUser WITH PASSWORD = 'YourStrongPassword123!';
GO

USE [SmartLogDb];
GO

CREATE USER SmartLogUser FOR LOGIN SmartLogUser;
GO

ALTER ROLE db_owner ADD MEMBER SmartLogUser;
GO
Important: Replace YourStrongPassword123! with a strong, unique password. Save this password — you'll need it for configuration.

3 Enable SQL Server Authentication

If using the SQL login created above:

  1. In SSMS, right-click the server → Properties
  2. Go to Security page
  3. Select "SQL Server and Windows Authentication mode"
  4. Click OK and restart the SQL Server service

5. Configure the Application

5.1 Environment Variables

SmartLog uses environment variables for sensitive configuration. Set these in System Environment Variables:

  1. Press Win + R, type sysdm.cpl, press Enter
  2. Go to Advanced tab → Environment Variables
  3. Under System variables, click New and add each variable:
Variable NameValueDescription
SMARTLOG_DB_CONNECTION Server=localhost\SQLEXPRESS;Database=SmartLogDb;User Id=SmartLogUser;Password=YourStrongPassword123!;TrustServerCertificate=True; SQL Server connection string
SMARTLOG_HMAC_SECRET_KEY (see below) Secret key for QR code signing
SMARTLOG_SEED_PASSWORD Your admin password Password for the default admin accounts
ASPNETCORE_ENVIRONMENT Production Set to Production for deployment

Generate HMAC Secret Key

Open PowerShell and run this to generate a secure random key:

# Generate a 64-character random HMAC secret
$bytes = New-Object byte[] 32
[System.Security.Cryptography.RandomNumberGenerator]::Fill($bytes)
[Convert]::ToBase64String($bytes)

Copy the output and use it as the SMARTLOG_HMAC_SECRET_KEY value.

Save this HMAC secret key! You will need the exact same key when configuring the Scanner App. If you lose it, all existing QR codes will become invalid and need to be regenerated.

Admin Password Requirements

The SMARTLOG_SEED_PASSWORD must meet these requirements:

  • At least 8 characters
  • At least 1 uppercase letter
  • At least 1 lowercase letter
  • At least 1 digit
  • At least 1 special character (e.g., !@#$%)
After setting environment variables, close and reopen your Command Prompt / PowerShell window for them to take effect.

5.2 Application Settings (Optional)

The file src/SmartLog.Web/appsettings.json contains additional settings. You typically don't need to modify this file — most settings are configured via the admin UI or environment variables.

SettingDefaultDescription
Sms:EnabledtrueEnable/disable SMS notifications globally
Sms:DefaultProviderGSM_MODEMSMS gateway: GSM_MODEM or SEMAPHORE
Sms:GsmModem:PortNameCOM3Serial port for GSM modem (check Device Manager)

6. Build & Run the Application

1 Build the Project

cd C:\SmartLogWebApp
dotnet build

You should see Build succeeded with 0 errors.

2 Run the Application

dotnet run --project src/SmartLog.Web --urls "https://0.0.0.0:8081;http://0.0.0.0:8080"

On first run, the application will:

  • Apply all database migrations (create tables automatically)
  • Seed default roles (SuperAdmin, Admin, Teacher, Security, Staff)
  • Create default admin accounts using SMARTLOG_SEED_PASSWORD
  • Seed grade levels, academic years, sections, and SMS templates

You should see output like:

[INF] Starting SmartLog Web Application
[INF] Applying database migrations...
[INF] Database migrations applied successfully
[INF] Created default admin user: admin.amy
[INF] Created default super admin user: super.admin

3 Verify It's Running

Open a web browser on the same machine and go to:

http://localhost:8080

You should see the SmartLog login page.

The application is running! Continue to the next sections to configure networking so other devices on the school LAN can access it.

8. Network Configuration (Static IP)

The server needs a static IP address so scanner devices can reliably connect to it on the school LAN.

1 Find Your Current Network Settings

Open Command Prompt and run:

ipconfig

Note these values for your Ethernet adapter (not Wi-Fi):

  • IPv4 Address (e.g., 192.168.1.100)
  • Subnet Mask (e.g., 255.255.255.0)
  • Default Gateway (e.g., 192.168.1.1)

2 Set a Static IP Address

  1. Press Win + R, type ncpa.cpl, press Enter
  2. Right-click your Ethernet adapter → Properties
  3. Select Internet Protocol Version 4 (TCP/IPv4) → Properties
  4. Select "Use the following IP address"
  5. Enter:
    FieldExample Value
    IP address192.168.1.100
    Subnet mask255.255.255.0
    Default gateway192.168.1.1
    Preferred DNS8.8.8.8
    Alternate DNS8.8.4.4
  6. Click OK → Close
Tip: Choose an IP address outside your router's DHCP range to avoid conflicts. Common choices: 192.168.1.100 – 192.168.1.199. Check your router's admin page for the DHCP range.

3 Verify Network Connectivity

:: Test internet
ping 8.8.8.8

:: Test from another device on the LAN
:: On another computer, open Command Prompt:
ping 192.168.1.100

9. Firewall Configuration

Windows Firewall blocks incoming connections by default. You need to allow traffic on the ports SmartLog uses.

1 Open Firewall Ports

Open PowerShell as Administrator and run:

# Allow HTTP (port 8080)
New-NetFirewallRule -DisplayName "SmartLog HTTP" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow

# Allow HTTPS (port 8081)
New-NetFirewallRule -DisplayName "SmartLog HTTPS" -Direction Inbound -Protocol TCP -LocalPort 8081 -Action Allow

2 Verify Access from Another Device

On another computer or phone connected to the same LAN, open a browser and go to:

http://192.168.1.100:8080

Replace 192.168.1.100 with your server's static IP. You should see the SmartLog login page.

Security Note: Only open these ports on the school LAN. Do NOT expose these ports to the internet unless you have proper SSL certificates and a reverse proxy (e.g., IIS or Nginx) configured.

10. Connect Scanner Devices

Scanner devices (running the SmartLog Scanner App) connect to the web app via the school LAN.

1 Register a Scanner Device

  1. Log in to SmartLog Web as an Admin
  2. Navigate to Admin → Device Management → Register Device
  3. Enter a device name and location (e.g., "Main Gate Scanner")
  4. Click Register
  5. Copy the API key shown — this is only displayed once!
Save the API key immediately! It is only shown once during registration. If you lose it, you'll need to regenerate a new one.

2 Configure the Scanner App

On the scanner device (Mac or Windows), open the SmartLog Scanner App and enter:

FieldValue
Server URLhttp://192.168.1.100:8080 (your server's static IP)
API KeyThe API key from device registration
HMAC SecretThe same SMARTLOG_HMAC_SECRET_KEY from the server
Accept Self-Signed CertsEnable if using HTTPS with self-signed certificate

3 Test the Connection

In the Scanner App setup, click Test Connection. You should see a success message confirming the scanner can reach the server.

11. Run as a Windows Service (Production)

For production, run SmartLog as a Windows Service so it starts automatically on boot and runs in the background.

1 Publish the Application

cd C:\SmartLogWebApp
dotnet publish src/SmartLog.Web -c Release -o C:\SmartLogPublish

2 Create the Windows Service

Open PowerShell as Administrator:

sc.exe create SmartLogWeb binPath="C:\SmartLogPublish\SmartLog.Web.exe --urls \"https://0.0.0.0:8081;http://0.0.0.0:8080\"" start=auto DisplayName="SmartLog Web Application"

# Set the service description
sc.exe description SmartLogWeb "SmartLog School Information Management System - Web Application"

3 Start the Service

# Start the service
sc.exe start SmartLogWeb

# Verify it's running
sc.exe query SmartLogWeb

The service will now start automatically when Windows boots.

Managing the Service

# Stop the service
sc.exe stop SmartLogWeb

# Restart after updates
sc.exe stop SmartLogWeb
dotnet publish src/SmartLog.Web -c Release -o C:\SmartLogPublish
sc.exe start SmartLogWeb

# Remove the service (if needed)
sc.exe delete SmartLogWeb

12. HTTPS / SSL Certificate Setup

Option A: Self-Signed Certificate (Development / Internal LAN)

The .NET development certificate works for testing but will show browser warnings:

dotnet dev-certs https --trust

Option B: Generate a Self-Signed Certificate for Production LAN

Open PowerShell as Administrator:

# Generate a self-signed certificate valid for 3 years
$cert = New-SelfSignedCertificate `
    -DnsName "192.168.1.100","smartlog.local" `
    -CertStoreLocation "Cert:\LocalMachine\My" `
    -NotAfter (Get-Date).AddYears(3) `
    -FriendlyName "SmartLog Web Server"

# Export the thumbprint (you'll need this)
$cert.Thumbprint

Then configure the app to use it by adding to appsettings.Production.json:

{
  "Kestrel": {
    "Endpoints": {
      "Https": {
        "Url": "https://0.0.0.0:8081",
        "Certificate": {
          "Subject": "192.168.1.100",
          "Store": "My",
          "Location": "LocalMachine"
        }
      },
      "Http": {
        "Url": "http://0.0.0.0:8080"
      }
    }
  }
}
When using self-signed certificates, enable "Accept Self-Signed Certs" in the Scanner App setup.

13. Backup & Maintenance

Database Backup

Schedule regular database backups using SQL Server Agent or a simple batch script:

:: backup-smartlog.bat — Run as scheduled task daily
@echo off
set BACKUP_DIR=C:\SmartLogBackups
set TIMESTAMP=%date:~-4%%date:~3,2%%date:~0,2%_%time:~0,2%%time:~3,2%
set TIMESTAMP=%TIMESTAMP: =0%

sqlcmd -S localhost\SQLEXPRESS -Q "BACKUP DATABASE [SmartLogDb] TO DISK='%BACKUP_DIR%\SmartLogDb_%TIMESTAMP%.bak' WITH COMPRESSION"

:: Delete backups older than 30 days
forfiles /p "%BACKUP_DIR%" /m *.bak /d -30 /c "cmd /c del @path" 2>nul

To schedule this:

  1. Open Task Scheduler (taskschd.msc)
  2. Create a new task that runs backup-smartlog.bat daily at midnight
  3. Run whether user is logged on or not

Application Updates

:: 1. Stop the service
sc.exe stop SmartLogWeb

:: 2. Pull latest changes
cd C:\SmartLogWebApp
git pull

:: 3. Rebuild and publish
dotnet publish src/SmartLog.Web -c Release -o C:\SmartLogPublish

:: 4. Start the service
sc.exe start SmartLogWeb

Log Files

Serilog writes logs to the console output. When running as a service, check the Windows Event Viewer or configure file logging in appsettings.json:

"Serilog": {
  "WriteTo": [
    { "Name": "Console" },
    {
      "Name": "File",
      "Args": {
        "path": "C:\\SmartLogLogs\\smartlog-.log",
        "rollingInterval": "Day",
        "retainedFileCountLimit": 31
      }
    }
  ]
}

14. Troubleshooting

Application Won't Start

ErrorSolution
Connection string not found Set the SMARTLOG_DB_CONNECTION environment variable. Restart the terminal.
SMARTLOG_SEED_PASSWORD is required Set the SMARTLOG_SEED_PASSWORD env var (required when ASPNETCORE_ENVIRONMENT=Production).
Cannot connect to SQL Server Verify SQL Server is running. Check the connection string. Ensure TCP/IP is enabled in SQL Server Configuration Manager.
Port already in use Another app is using port 8080/8081. Find it: netstat -ano | findstr :8080 and stop it, or use different ports.

Scanner Can't Connect

IssueSolution
Connection refused Check firewall rules. Verify the server URL includes the port (e.g., http://192.168.1.100:8080).
SSL/TLS error Enable "Accept Self-Signed Certs" in the Scanner App if using self-signed certificates.
API key rejected Verify the API key matches what was generated during device registration. Keys are case-sensitive.
HMAC validation failed The SMARTLOG_HMAC_SECRET_KEY on the server must match the HMAC Secret in the Scanner App settings.
Scanner sees "offline" Ping the server IP from the scanner device. Check that GET http://SERVER_IP:8080/api/v1/health returns OK.

Database Issues

IssueSolution
Migration errors Try: dotnet ef database update -p src/SmartLog.Web
Database is full SQL Server Express has a 10 GB limit. Archive old scan data or upgrade to Standard edition.
