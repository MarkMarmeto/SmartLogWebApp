## User Story

**As a** System
**I want** to validate QR code signatures using HMAC-SHA256
**So that** tampered or forged QR codes are rejected

## Context

### Technical Context
QR codes contain an HMAC signature that must be verified server-side to ensure the code was generated by the system and not tampered with.

---

## Acceptance Criteria

### AC1: Parse QR Payload
- **Given** a QR payload `SMARTLOG:STU-2026-001:1706918400:abc123hmac`
- **When** the system parses it
- **Then** it extracts:
  - Prefix: "SMARTLOG"
  - StudentId: "STU-2026-001"
  - Timestamp: 1706918400 (Unix epoch)
  - Signature: "abc123hmac" (Base64 encoded)

### AC2: Validate Prefix
- **Given** a QR payload
- **When** the prefix is not "SMARTLOG"
- **Then** validation fails with error "Invalid QR format"

### AC3: Validate HMAC Signature
- **Given** a parsed QR payload
- **When** the system computes HMAC-SHA256 of `{studentId}:{timestamp}` using the secret key
- **Then** the computed signature must match the provided signature
- **And** if mismatch, validation fails with error "Invalid signature"

### AC4: Lookup QR Code Record
- **Given** a valid HMAC signature
- **When** the system looks up the QR code in the database
- **Then** it finds the record by StudentId and Payload
- **And** verifies IsValid = true

### AC5: Reject Invalidated QR Codes
- **Given** a QR code that has been regenerated (IsValid = false)
- **When** it is scanned
- **Then** validation fails with error "QR code invalidated"
- **And** the old QR code should not authenticate the student

### AC6: Return Student Information
- **Given** QR validation succeeds
- **Then** return the associated student record:
  - StudentId
  - FirstName, LastName
  - Grade, Section
  - IsActive status

### AC7: Constant-Time Comparison
- **Given** HMAC signature comparison
- **Then** use constant-time comparison to prevent timing attacks

---

## Edge Cases & Error Handling

| Scenario | Expected Behaviour |
|----------|-------------------|
| Malformed QR payload | Fail: "Invalid QR format" |
| Wrong number of segments | Fail: "Invalid QR format" |
| Invalid Base64 signature | Fail: "Invalid signature encoding" |
| QR code not in database | Fail: "QR code not found" |
| Student deleted | Fail: "Student not found" |
| HMAC key not configured | Server error (log critical) |
| Very old timestamp | Accept (timestamp is for audit, not expiry) |

## Technical Notes

### HMAC Computation
```csharp
public bool ValidateQrCode(string payload, out ValidationResult result)
{
    // Parse: SMARTLOG:{studentId}:{timestamp}:{signature}
    var parts = payload.Split(':');
    if (parts.Length != 4 || parts[0] != "SMARTLOG")
    {
        result = ValidationResult.InvalidFormat;
        return false;
    }

    var studentId = parts[1];
    var timestamp = parts[2];
    var providedSignature = parts[3];

    // Compute expected HMAC
    var dataToSign = $"{studentId}:{timestamp}";
    var expectedSignature = ComputeHmacSha256(dataToSign, _secretKey);

    // Constant-time comparison
    if (!CryptographicOperations.FixedTimeEquals(
        Convert.FromBase64String(providedSignature),
        Convert.FromBase64String(expectedSignature)))
    {
        result = ValidationResult.InvalidSignature;
        return false;
    }

    // Lookup in database...
}
```
