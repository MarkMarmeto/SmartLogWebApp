Step 1: Clone the Repository

Open PowerShell or Command Prompt and clone the repository:

cd C:\Projects
git clone https://github.com/MarkMarmeto/SmartLogWebApp.git
cd SmartLogWebApp
If you don't have Git, you can download the source code as a ZIP file from GitHub and extract it to C:\Projects\SmartLogWebApp.

Step 2: Run the Setup Wizard

The automated setup wizard handles everything: database creation, security configuration, building, firewall rules, and Windows Service installation.

Option A: Using the .bat launcher (recommended)

Right-click deploy\Setup-SmartLog.bat → Run as administrator

Option B: Using PowerShell directly

powershell -NoProfile -ExecutionPolicy Bypass -File .\deploy\Setup-SmartLog.ps1
Must run as Administrator. The script needs admin privileges to create the Windows Service, set system environment variables, and configure firewall rules.

Setup Steps Explained

The wizard runs through 9 steps interactively:

1Check Prerequisites

Verifies .NET SDK 8.0+, Git, and SQL Server are installed and running.

2Database Configuration

Prompts for SQL Server instance name, database name, and username. Can auto-generate a secure database password.

SettingDefault
SQL Instancelocalhost\SQLEXPRESS
Database NameSmartLogDb
Database UserSmartLogUser
3Create Database & SQL Login

Creates the database and SQL login using Windows Authentication. Enables mixed-mode authentication and optionally restarts SQL Server to apply the change.

4Security Configuration

Configures the HMAC secret key (for QR code signing) and admin password.

  • HMAC Key: Can auto-generate a 32-byte Base64 key or enter an existing one
  • Admin Password: Must meet complexity requirements (8+ chars, upper, lower, digit, special)
Save the HMAC key! You will need to enter the same key in the SmartLog Scanner App. This key is shown only once during setup.
5Network Configuration

Prompts for the HTTP port (default: 8080). Detects the machine's IP address for LAN access.

6Set Environment Variables

Sets system-wide environment variables for the database connection, HMAC key, admin password, and ASP.NET Core URLs.

7Build & Publish

Runs dotnet publish in Release mode, outputting to C:\SmartLog.

8Configure Firewall

Creates an inbound firewall rule for the HTTP port on Domain and Private networks.

9Install Windows Service

Registers SmartLog as a Windows Service with auto-start on boot and automatic restart on failure (5s, 10s, 30s intervals).

Step 3: Verify Installation

  1. Open a browser and navigate to http://localhost:8080 (or your configured port)
  2. You should see the SmartLog login page
  3. Log in with super.admin and the password you set during setup
  4. You will be prompted to change your password on first login
If you see the login page, the installation is complete. Configure the Scanner App with the same HMAC key to start scanning.

Updating SmartLog

When bug fixes, security patches, or new features are available, use the update script to safely pull and deploy changes.

What the update script does:
  1. Validates the existing installation (service, install directory, repo)
  2. Checks for updates via git fetch and shows new commits
  3. Backs up the current build to C:\SmartLogBackups\ (timestamped)
  4. Stops the Windows Service
  5. Pulls the latest code from GitHub
  6. Builds & publishes the new version to C:\SmartLog\
  7. Auto-rolls back from backup if the build fails
  8. Starts the service and verifies it's responding
Current State Update Process Result ───────────── ────────────── ────── C:\SmartLog\ ──backup──▶ C:\SmartLogBackups\ Preserved (running v1.0) smartlog-backup-2026-03-22\ ──stop────▶ Service stopped ──pull────▶ git pull origin main ──build───▶ dotnet publish → C:\SmartLog\ │ ├── Success → Start service ──▶ Running v1.1 │ └── Failure → Restore backup Start service ──▶ Running v1.0 (safe rollback)

Running the Update

Option A: Using the .bat launcher (recommended)

Right-click deploy\Update-SmartLog.bat → Run as administrator

Option B: Using PowerShell directly

powershell -NoProfile -ExecutionPolicy Bypass -File .\deploy\Update-SmartLog.ps1

Optional parameters

ParameterDescriptionDefault
-SkipBackupSkip creating a backup before updatingfalse
-BranchGit branch to pull frommain
powershell -ExecutionPolicy Bypass -File .\deploy\Update-SmartLog.ps1 -Branch develop
Downtime: The app will be unavailable for 30-60 seconds during the update (service stop → build → service start). Plan updates during off-hours if possible.

Manual Rollback

If something goes wrong after an update, you can manually restore a previous version:

REM 1. Stop the service
sc.exe stop SmartLogWeb

REM 2. Restore from backup (replace timestamp with actual backup folder)
xcopy /E /Y "C:\SmartLogBackups\smartlog-backup-2026-03-22_143000\*" "C:\SmartLog\"

REM 3. Start the service
sc.exe start SmartLogWeb

The update script keeps the last 5 backups. View available backups:

dir C:\SmartLogBackups\

Generating an HMAC Secret Key

The HMAC secret key is used to cryptographically sign student QR codes. Both the web app and scanner app must use the same key for QR code validation to work.

Option A: Using the GenerateHMAC script (recommended)

Right-click deploy\GenerateHMAC.bat → Run as administrator

Or via PowerShell:

powershell -NoProfile -ExecutionPolicy Bypass -File .\deploy\GenerateHMAC.ps1

The script generates a cryptographically secure 32-byte Base64-encoded key and copies it to your clipboard.

Option B: Using PowerShell one-liner

$k = New-Object byte[] 32; [System.Security.Cryptography.RandomNumberGenerator]::Fill($k); [Convert]::ToBase64String($k)

Option C: Auto-generated during Setup

The Setup-SmartLog.ps1 wizard can auto-generate the key during initial installation (Step 4).

Updating the HMAC Key

Warning: Changing the HMAC key will invalidate ALL existing student QR codes. Students will need new QR codes printed after a key change.

There are two ways to update the HMAC key:

Method 1: Via Admin Settings UI

  1. Log in as Admin or SuperAdmin
  2. Go to Admin > Settings > QRCode tab
  3. Update the HmacSecretKey field
  4. Click Save
  5. All existing QR codes will be automatically invalidated
  6. Regenerate QR codes for students from the Students page

Method 2: Via Environment Variable

REM Set the new key
setx /M SMARTLOG_HMAC_SECRET_KEY "your-new-base64-key-here"

REM Restart the service
sc.exe stop SmartLogWeb
sc.exe start SmartLogWeb
Remember: After changing the HMAC key on the web app, you must also update it on the Scanner App via the Setup Wizard so both apps use the same key.

Logs & Troubleshooting

Log Location

C:\SmartLog\logs\

View recent logs

Get-Content C:\SmartLog\logs\*.log -Tail 50

Common Issues

Service fails to start
  • Check that SQL Server is running: sc.exe query MSSQL$SQLEXPRESS
  • Verify environment variables are set: set SMARTLOG
  • Check logs: Get-Content C:\SmartLog\logs\*.log -Tail 100
  • Check Windows Event Viewer: Application log for SmartLogWeb errors
Scanner QR codes rejected (Invalid HMAC)
  • Verify HMAC keys match on both web app and scanner app
  • Check web app key: Admin > Settings > QRCode > HmacSecretKey
  • Check scanner key: Setup Wizard > HMAC Secret field
  • If keys were changed, regenerate student QR codes
PowerShell script won't run (execution policy)

Use the .bat launcher files instead, or run with:

powershell -ExecutionPolicy Bypass -File .\script.ps1

Or set the policy permanently:

Set-ExecutionPolicy RemoteSigned -Scope LocalMachine
Database connection failed
  • Ensure SQL Server service is running
  • Verify TCP/IP is enabled in SQL Server Configuration Manager
  • Check that mixed-mode authentication is enabled
  • Test connection: sqlcmd -S localhost\SQLEXPRESS -U SmartLogUser -P "password"

Backup & Recovery

Application Backups

The update script automatically backs up to C:\SmartLogBackups\ before each update. Last 5 backups are kept.

Database Backups

Use SQL Server Management Studio (SSMS) or the command line to back up your database:

sqlcmd -S localhost\SQLEXPRESS -Q "BACKUP DATABASE [SmartLogDb] TO DISK='C:\SmartLogBackups\SmartLogDb.bak' WITH FORMAT"
Application backups do NOT include the database. Back up your database separately before major updates.

Network Configuration

SmartLog runs on a local network (LAN). The server PC and all scanner devices must be on the same network. A static IP address is required so the server address doesn't change when the PC restarts.

Automated Setup

Run the network helper script to auto-detect your network and configure a static IP:

Setup-Network.bat    (right-click > Run as administrator)

The script will:

  1. Detect all active network adapters and current settings
  2. Calculate your available IP address range
  3. Scan for unused IPs and suggest one
  4. Optionally apply the static IP configuration

Understanding IP Ranges

First, open a command prompt and run ipconfig to see your current network:

Ethernet adapter Ethernet:
   IPv4 Address. . . . . . . : 192.168.2.150
   Subnet Mask . . . . . . . : 255.255.255.0
   Default Gateway . . . . . : 192.168.2.1

Your IP range is determined by the Subnet Mask:

Subnet MaskNetwork PrefixAvailable IP RangeTotal Hosts
255.255.255.0 (/24)First 3 octets fixed192.168.2.1 - 192.168.2.254253
255.255.0.0 (/16)First 2 octets fixed192.168.0.1 - 192.168.255.25465,533
255.0.0.0 (/8)First octet fixed10.0.0.1 - 10.255.255.25416 million+

Common private network ranges:

RangeTypical UseExample
192.168.x.xHome/small office routers192.168.1.100, 192.168.2.10
10.x.x.xSchool/enterprise networks10.0.1.50, 10.10.0.100
172.16.x.x - 172.31.x.xMedium-sized networks172.16.0.50

Choosing a Static IP

Based on your ipconfig output:

  1. The static IP must be in the same network as your gateway (matching the first 3 octets for /24 networks)
  2. Pick an IP in the low range (10-50) to avoid conflicts with DHCP, which typically assigns from 100+
  3. Avoid .1 (usually the router/gateway) and .255 (broadcast address)
Example: If your gateway is 192.168.2.1 and subnet is 255.255.255.0:
Good static IPs: 192.168.2.10, 192.168.2.20, 192.168.2.30
The gateway (192.168.2) must match. Only the last number changes.

Manual Static IP Setup (Windows)

  1. Open Settings > Network & Internet > Ethernet
  2. Click on your active network connection
  3. Click Edit next to IP assignment
  4. Change from Automatic (DHCP) to Manual
  5. Enable IPv4 and enter:
    IP addressYour chosen static IP (e.g., 192.168.2.10)
    Subnet maskSame as ipconfig output (e.g., 255.255.255.0)
    GatewaySame as ipconfig output (e.g., 192.168.2.1)
    Preferred DNSSame as gateway or 8.8.8.8 (Google DNS)
  6. Click Save

Verify Network Configuration

After setting the static IP:

  1. Verify the IP is applied: ipconfig
  2. Check internet access: ping google.com
  3. Verify firewall rules allow inbound traffic on port 8080
  4. Access SmartLog from the server: http://localhost:8080
  5. Access from another device on the same network: http://[static-ip]:8080
  6. Test health endpoint from scanner device: http://[static-ip]:8080/api/v1/health
Tip: If the server has multiple network adapters (Wi-Fi + Ethernet), ensure the static IP is set on the adapter connected to the same network as the scanner devices. Use Ethernet for best reliability.

Scanner App Configuration

In the Scanner App setup wizard, enter the server URL as:

http://[static-ip]:8080

Example: http://192.168.2.10:8080

All scanner devices on the same LAN will connect to this address.
